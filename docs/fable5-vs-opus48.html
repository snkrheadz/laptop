結論

決定的な違いは 「ZDR(ゼロデータ保持)非対応」 と 「料金が正確に2倍」 の2点。最高性能が必要でZDR不要・コスト2倍を許容できる場合のみ Fable 5。ZDR運用中・高速応答重視・コスト効率重視なら Opus 4.8 が無難。

Fable 5 — 最高性能 / $10·$50 / ZDR不可 / 30日保持 Opus 4.8 — 高効率 / $5·$25 / ZDR可 / Fast mode対応

02 各観点の詳細

5つの観点ごとの検証結果

① INPUT / OUTPUT 料金

Fable 5 は 入力 $10 / 出力 $50 で、Opus 4.8($5 / $25)の正確に2倍。両モデルとも Opus 4.7 世代の同一トークナイザを使うため、同一テキストのトークン換算に差はなく、実質コストもそのまま2倍になります。

② 速度(latency)

公式の「Comparative latency」表に Fable 5 の格付けは掲載されていません(Opus 4.8 は "Moderate")。Fable 5 は最も要求の厳しい推論・長期エージェント作業向けの最上位モデルかつ Adaptive thinking 常時ON のため、体感速度は Opus 4.8 と同等かそれ以下(遅め)と見るのが現実的です。

公式に数値の相対評価がないため、断定は避け推定として記載しています。

④ 利用規約・学習利用

商用入力の学習利用は両者とも「無断では行わない」(Commercial Terms 上、モデル別の例外なし・一律)。Usage Policy も共通です。

Fable 5 は安全分類器を内蔵し、拒否時は refusal(HTTP 200)+ fallbacks パラメータで別モデルへ自動リトライ可能。出力前に拒否された場合は課金なしです。

03 ZDR・データポリシー(最重要)

Fable 5 は "Covered Model" に指定されている

⚠ Fable 5 への切り替えは ZDR 運用と両立しない

Fable 5 / Mythos 5 は "Covered Models" に指定され、30日間のデータ保持が必須。ZDR(ゼロデータ保持)契約の対象外です。データ保持設定が要件を満たさない組織からのリクエストは 400 invalid_request_error を返します。

  • Opus 4.8 は Covered Model ではなく、Messages API は ZDR 適格。
  • 既に ZDR 契約で運用中の組織が Claude Code の Model を Fable 5 に切り替えると、入出力が30日保持される(またはリクエスト自体が拒否される)。
  • ZDR はそもそも商用組織のAPIキー / Claude Enterprise 経由のみ適格。Pro / Max などコンシューマ製品は元々 ZDR 対象外。
  • コンプライアンス要件次第では Fable 5 への切替自体が不可になり得る最重要ポイント。

04 Claude Code 固有の注意点

Model 設定として使う際の実務的な差

  • Fast mode は Opus 4.8 / 4.7 / 4.6 のみ対応。Fable 5 は非対応。高速応答を重視するなら Opus 4.8 が有利。
  • effort は両モデルとも Claude Code 上で既定 high。
  • Fable 5 は Adaptive thinking 常時ON、生の思考内容(CoT)は非返却(既定 omitted)。
  • サブスクリプション(Pro / Max)経由の場合、課金は上記 API 単価ではなくプランの利用上限に従う。表中の $ 単価は APIキー直叩き(従量)時の比較。
